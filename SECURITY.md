# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ |
| < 1.0   | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please follow responsible disclosure:

1. **Do NOT open a public GitHub issue** for security vulnerabilities.
2. Email the maintainer directly at: **intersysclaude@gmail.com**
   - Subject: `[SECURITY] <repo-name> — Brief Description`
3. Include:
   - A description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

I will acknowledge receipt within **48 hours** and aim to provide a fix or mitigation within **14 days** for critical issues.

## Scope

This policy applies to the source code in this repository. Third-party dependencies
are out of scope — please report those to the respective maintainers.

## Responsible Disclosure

I am committed to working with security researchers in good faith. I will:
- Acknowledge your report promptly
- Keep you informed of remediation progress
- Credit you in the release notes (unless you prefer to remain anonymous)

Thank you for helping keep this project and its users safe.
